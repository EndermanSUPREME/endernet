    C2 Server using C++

    I was getting interested in the various C2 Clients such as the famous HAVOC and Sliver part of me wondered, "could I possibly build my own C2 Server?" I knew I wouldn't have anything as spectacular as the two I just mentioned but I figured the experience and learning opportunity would be great to have, so I went for it. I figured the main gist of the C2 I needed, was the ability to capture reverse-shells and save the sessions in some storage where I could access them later, I'm used to this functionality from using Meterpreter when doing Windows based CTFs.

    My solution that I implemented was to use a Socket Server and storing the newly connected sockets into a vector that I could easily move around in later, I had not tested it within a HTB Environment, but general testing showed the project to work ok, it would be much better such as faster speeds and so forth but I'm proud of it.
